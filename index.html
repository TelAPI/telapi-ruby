InboundXML

We've made generating InboundXML with this library as intuitive as possible by defining a simple code design pattern which corresponds to the structure of the XML. The best way to understand this pattern is to take a look at an example:

require 'telapi'
 
ix = Telapi::InboundXml.new
 
puts ix.response

/*
</Response>
*/

First and foremost, you need to instantiate a new InboundXml object. This assignment alone begins the process of generating the xml. As shown above, if we print ix.response, the start of the base elements required for an InboundXML document are shown.

require 'telapi'

ix = Telapi::InboundXml.new do
  Record(:action => "http://foo-url.com")
end
 
puts ix.response


/*
<Response>
  <Record action="http://foo-url.com/" />
</Response>
*/

Creating elements is super easy. Simply inovke the element you would like to add to the XML document within a block as shown above. Any desired attributes of the element are passed along when peforming the call.

require 'telapi'
 
ix = Telapi::InboundXml.new do
  Record(:action => "http://foo-url.com")
  Say("Hello!", :voice => "man")
end
 
puts ix.response

/*
<Response>
  <Record action="http://foo-url.com/" />
  <Say voice="man">Hello!<Say />
</Response>
*/

This dot chaining can continue for as many elements as needed in the response. Specifying of element attributes is achieved by passing them to to the element method. For details on element methods and which parameters are required when, see the InboundXML source here.

require 'telapi'
 
ix = Telapi::InboundXml.new do
  Record(:action => "http://foo-url.com")
  Say("Hello!", :voice => "man")
  Dial() { Coference("ConfRoom") }
end
 
puts ix.response


/*
<Response>
  <Record action="http://foo-url.com/" />
  <Say voice="man">Hello!<Say />
  <Dial>
    <Conference>ConfRoom<Conference/>
  <Dial />
</Response>
*/

Children elements, such as Conference, can be added to the document by putting them within a second block when calling the desired parent element. An example of this is shown above.

REST API

Before using any of the TelAPI REST methods available, you must authenticate your account. Do so by including your REST API credentials in the following way:

Telapi.config do |config|
  config.account_sid = '{AccountSid}'
  config.auth_token  = '{AuthToken}'
end

Notice the code above instantiates and assigns a Telapi object. Every method used throughout this helper library will use this main Telapi class.

Below are usage examples and additional information for each function of TelAPI accomplished via REST requests. If you need to dig deeper, the source defining each REST method is found here.

Accounts

View Account

Documentation

Request an account resource with the get method:

account = Telapi::Account.get

puts account

Note how any available properties of the account object can be accessed via dot notation:

puts account.sid
puts account.status

View Usage

Documentation

Request a usage resource with the get method:

usage = Telapi::Usage.get({sid})

p usage

Then simply use dot notation to access individual resource properties:

p usage.quantity
p usage.product

List Usages

Documentation

Request usages list resource with the list method:

usages = Telapi::Usage.list()

p usages

p call.price

List Calls

Documentation

Request a call list resource with the list method:

calls = Telapi::Call.list()
 
p calls

Make Call

Documentation

Make a call with the make method:

call = Telapi::Call.make("7325551234", "2095555678", "http://foo.com")
 
p call.sid

Interrupt Call

Documentation

Interrupt a call with the interrupt method:

Telapi::Call.interrupt({CallSid}, :Status' => 'Canceled')

Hangup Call

Documentation

Hangup a call with the hangup method:

Telapi::Call.hangup({CallSid})

Send Digits to Call

Documentation

To play DTMF digits during a call, use the send_digits method

Telapi::Call.send_digits( {CallSid}, '1234' )

Play Audio to Call

Documentation

To play audio to a call, use the play_audio method:

Telapi::Call.play_audio('{CallSid}', AudioUrl = 'http://www.example.com/sample.mp3')

Call Voice Effects

Documentation

To adjust call voice effects, use the voice_effect method:

Telapi::Call.play_audio( {CallSid}, :Pitch => '2' )

Record Call

Documentation

To record a call, use the record method:

Telapi::Call.record( {CallSid}, Record = 'true' )

List Participants

Documentation

Request a participant list resource with the list method:

Mute, Deaf, or Hangup Participant

The various actions which can be peformed on a participant (mute/deaf, hangup) can be invoked by simply appending them to the end of the participant object:

participant = Telapi::Participant.get({ConferenceSid}, {CallSid})

participant.mute
participant.unmute
participant.deaf
participant.undeaf
participant.hangup

Play Audio to Participant

Documentation

Play audio to a participant with the play_audio method:

participant = Telapi::Participant.get({ConferenceSid}, {CallSid})

participant.play_audio('http://www.telapi.com/audio.mp3')
